UNION

SQL语句中的UNION运算符用于连接两个SELECT语句的结果集。

UNION的主要作用和特征如下:

  • 用于连接两个SELECT语句的查询结果。
  • 两个SELECT语句必须包含相同数量的列。
  • 列的数据类型也应该兼容,如数值和字符串。
  • 连接后的结果集不包含重复的数据行。

sql注入解释

​ SQL 注入就是指 Web 应用程序对用户输入的数据合法性没有过滤或者是判断，攻击者可以在Web应用程序中事先定义好的查询语句的结尾上添加额外的SQL语句，在管理员不知情的情况下实现非法操作，以此来实现欺骗数据库服务器执行非授权的任意查询，从而进一步得到相应的数据信息。

条件

  • 传递给后端的参数是可以控制的

  • 参数内容会被带入到数据库查询

  • 变量未存在过滤或者过滤不严谨

宽字节注入

原理：mysql在使用 GBK 编码的时候，会认为两个字符为一个汉字， 例如%aa%5c 就是一个汉字（前一个ascii码大于128才能到汉 字的范围）。我们在过滤 ’ 的时候，往往利用的思路是将’ 转换 为 '

因此我们在此想办法将 ‘ 前面添加的 \ 除掉，一般有两种思路：
1、%df 吃掉 \ 具体的原因是 urlencode(‘) = %5c%27，我们在%5c%27 前面添加%df，形
成%df%5c%27，而上面提到的mysql在GBK编码方式的时候会将两个字节当做一个汉字，此事%df%5c就是一个汉字，%27则作为一个单独的符号在外面，同时也就达到了我们的目的。
2、将'中的\过滤掉，例如可以构造%**%5c%5c%27的情况，后面的%5c会被前面的%5c
给注释掉。这也是bypass的一种方法。
例子： Less-32，33，34，35，36，37

​ 1.正常情况下GPC开启或者使用addslashes函数过滤GET或POST提交的参数时，我们测试输入的’，就会被转义为';
​ 2.若存在宽字节注入，输入%df%27时，经过单引号的转义变成了%df%5c%27，之后再数据库查询语句进行GBK多字节编码，即一个中文占用两个字节，一个英文同样占用两个字节且在汉字编码范围内两个编码为一个汉字。然后MySQL服务器会对查询语句进行GBK编码即%df%5c转换成汉字”運”,单引号逃逸出来，从而绕过转义造成注入漏洞

WAF

WAF是Web应用程序防火墙的英文缩写(Web Application Firewall),它是一个应用在网络层或操作系统层的安全设备或软件。

作用：

  • 过滤恶意网络流量:

    识别和过滤恶意请求,如SQL注入、XSS攻击等。

  • 应用层防护:

    针对Web应用本身进行防护,比如URL、HTTP头部、请求内容等深度分析。

  • 防护Web服务:

    为Web服务器、Web应用程序提供安全防护,阻止常见攻击。

  • 日志记录:

    记录所有Web请求日志,包括正常和异常请求,便于后期分析。

  • WAF规则管理:

    支持自定义规则以识别更多攻击模式。

  • 动态防护:

    根据运行时行为实时学习并更新防护策略

工作模式：

Apache一共有3种稳定的MPM模式(MPM：多进程处理模块)，它们分别是 prefork、worker、event；

prefork 工作模式

Apache在启动之初， 就预先fork一些子进程， 然后等待请求进来。之所以这样做， 是为了减少频繁创建和销毁进程的开销。每个子进程只有一个线程， 在一个时间点内，只能处理一个请求。

  • 优点： 成熟稳定，兼容所有新老模块。同时， 不需要担心线程安全的问题。
  • 缺点： 一个进程相对占用更多的系统资源，消耗更多的内存。而且，它并不擅长处理高并发请求。

通俗解释

用更通俗直观的语言来解释Apache网页服务器工作原理:

Apache服务器就像一个24小时值守的门卫。它会一直监视着网站的大门,等待访客上门。

当有人来访时,门卫Apache会第一时间走出来迎接。它会问访客一些问题,比如姓名,来访目的等。这就是解析请求的过程。

门卫Apache会根据访客的要求,寻找网站内相关的文件或服务。就像帮助访客找到正确的人或部门。如果文件在网站内,它就会帮助访客取得文件。

门卫Apache还会检查访客的身份,是否有权限访问这些文件和服务。如果没有权限,就不允许访问。这是权限检查的过程。

取得文件后,门卫Apache会告诉访客一些基本信息,比如文件类型是什么。然后交给访客传阅。这是响应的过程。

整个访问结束后,门卫Apache会礼貌送访,并记录此次访问情况,以便日后参考。它还会及时应对更多访客,确保网站高效运行。

所以,Apache就像一个24小时值守网站的门卫,它会礼貌高效地为每一位访客提供服务,保证网站的正常访问。

二进制加法是一种计算二进制数之间加法的方法。二进制加法的规则与十进制加法很类似，只是在二进制数位上只允许有两个数值，0 和 1，因此每个位上的和最大只能是 1。

对于两个二进制数字 A 和 B 相加，可以按照以下步骤进行：

  1. 从最低位开始，将 A 和 B 的第一位相加，如果结果小于二进制数的基值 2，则直接写入和，否则将余数记为本位结果，进位 1 到下一位。
  2. 重复上述步骤，对每一位相加，直到加到最高位。
  3. 如果最高位有进位，则需要在结果最前面增加一个 1 表示进位。

因此，二进制数 1011 和 0110 的和为 10001（十进制数为 17）。

需要注意的是，二进制加法和十进制加法类似，可能会出现进位。如果最高位有进位，应该在结果的最前面添加一个数字 1 表示进位。此外，对于任意的二进制加法，如果两个二进制数位数不相等，可以在较短的二进制数的前面加上 0，使得它们的位数相等，然后再进行加法操作。

总之，二进制加法是计算二进制数之间加法的基本方法，其规则很简单，只需将两个二进制数对应位相加，并注意进位和最高位的处理即可。

路由表相关

每个路由器都有自己独立的路由表。

路由表是路由器内部维护的一个表,用于记录目的网络地址、下一跳路由器地址以及接口等信息,用于决定数据包的转发路径。

每个路由器的路由表内容是独立的,不会与其他路由器共享:

  • 路由表内容取决于路由器所处位置和邻居路由器信息。两台路由器可能位于不同网络位置,路由表内容不同。
  • 路由器通过不同的路由协议和静态路由配置学习和生成自己的路由表。
  • 即使同一供应商同型号路由器,如果网络位置和配置不同,路由表也会有差异。
  • 即使同一供应商路由器通过同一路由协议学习路由,由于网络拓扑和时延不同,两个路由表同步存在延迟。

所以总体来说:

  • 每个路由器都有自己独立的路由表
  • 路由表内容取决于路由器网络位置和配置信息
  • 不同路由器的路由表内容可能不同

这就保证了每个路由器都可以根据自身路由表正确转发数据包。

TCP协议

TCP是一种可靠的传输协议，在建立和终止连接时使用了三次握手和四次挥手的过程。

1.三次握手(Three-Way Handshake) :

a.第一步:客户端向服务器发送一个带有SYN(同步）标志的数据包，指示客户端要发起连接请求.

b.第二步:服务器收到客户端的请求后，回复一个带有SYN/ACK(同步/确认)标志的数据包，表示收到了请求，并同意建立连接。

c.第三步:客户端收到服务器的回复后，再发送一个带有ACK(确认）标志的数据包，表示已经收到服务器的同意，连接建立成功。

2四次挥手(Four-Way Handshake) :

a.第一步:当客户端要关闭连接时，发送一个带有FIN(结束）标志的数据包，表示不再发送数据。

b.第二步:服务器收到客户端的关闭请求后，回复一个带有ACK标志的数据包，表示已收到关闭请求。

c.第三步:服务器继续发送未完成的数据，将其发送给客户端。

d.第四步:服务器发送一个带有FIN标志的数据包，表示服务器的数据发送完毕，请求关闭连接。

e.第五步:客户端收到服务器的关闭请求后，回复一个带有ACK标志的数据包，表示已收到关闭请求。

f.第六步:客户端等待一段时间，确保服务器收到ACK回复后，关闭连接。

这样，经过三次握手和四次挥手的过程，TCP连接即建立和关闭完成。这些过程保证了可靠的数据传输和连接的安全关闭。

      • 子网的网络地址(很少情况)
      • 子网内第一个可用的主机地址
      • 子网的网关地址,路由器或交换机的IP地址
  • 一般来说,192.168.1.1最常见的情况是作为子网内第一个可用地址分配,也常作为子网的默认网关地址。

所以总结来说,192.168.1.1是一个私有IP地址,它可能代表单独主机的IP,也可能是子网内第一个地址或默认网关地址。

192.168.1.1/30是一个子网掩码为30位的子网。

子网掩码30位,表示:

  • 网络地址部分:192.168.1.0
  • 主机地址部分:2位(2^2=4个地址)

所以该子网可以分配的IP地址有:

  • 网络地址:192.168.1.0
  • 广播地址:192.168.1.3
  • 可用主机地址:192.168.1.1、192.168.1.2

总共可以分配2个IP地址给主机使用。

所以答案是:192.168.1.1/30子网可以分配2个主机IP地址。

a有8位，b有8位，c有8位，d应该有8位a+b+c+d=32

192.168.1.1/30 8+8+8+6=30,32-30=2,主机地址有两位（2^两=4个地址），分别减去网络地址(192.168.1.0)和广播地址(192.168.1.3)，可用地址剩下的是192.168.1.1、192.168.1.2

192.168.1.0不是一个合法的IP地址,它是网络地址,不能直接作为主机或网关使用。

正确的情况是:

  • 网络地址: 192.168.1.0
  • 第一个可用主机地址: 192.168.1.1
  • 网关地址: 192.168.1.1

理由如下:

  • 在IP地址分配中,网络地址是不可用的,它只表示子网本身。
  • 根据地址分配规则,第一个可用地址应该是网络地址加1,也就是192.168.1.1。
  • 网关地址一般设置为第一个可用主机地址,也就是192.168.1.1,以便路由转发。

所以您指出的正确,192.168.1.0是网络地址,不应该作为主机或网关使用。网关地址应该是第一个可用主机地址192.168.1.1。
