1.3-博客的编写与上传

2023-12-03

博客的编写与上传

1.准备准备步骤

下载Typora（https://typoraio.cn/）
➤下载PicGo（https://www.picgo.cn/）
PS：期间记得挂代理

➤自行安装Typora以及PicGo至相应位置。

2.打开Typora,进入设置界面的“偏好设置”

➤在“偏好设置”中选择图像，更改图像选项中的选项如图所示，上传服务选择PicGo（app），PicGo路径按照实际选择，路径最后应为PicGo.exe，这个时候先将Typora放在一边。

3.个人令牌获取

➤进入GitHub主页，进入设置界面，选择左边的开发者设置/ActionListItem-label。

➤选择个人访问令牌，进入令牌（经典），生成新的令牌，期间需要输入GitHub密码。

➤进入新建页面，备注根据自身需求来写（不影响后续，自己看得懂就行），有效期选择无有效期（每个人的汉化不同，表达可能不一样），即自定义下面的选项，选择作用域选择：repo，其他不用选。

➤生成访问令牌（在页面最下面），复制访问令牌，访问令牌后面需要用。

4.在GitHub中创建一个博客图片仓库，名字尽量明显易记。
5.PicGo设置

➤图床设置->GitHub->编辑一个选项

➤图床配置名自行设置（自己懂就行），设定仓库名为：GitHub用户名/博客图片仓库名，设定分支名为main，设定Token为个人访问令牌，设定存储路径为img/，自定义域名自行设置，确定。

6.验证图片上传选项

➤返回Typora偏好设置->图像中，点击验证图片上传选项，第一次验证需要额外确认一次，确认完成后，等待1~3分钟，再次确认，请确保最后弹出的窗体为验证成功。

7.写博客与上传图片

➤博客文字格式请参照markdown的使用说明

➤图片上传只需将图片复制至正在写的文章中，鼠标右键图片，选择上传图片，等待几秒即可。

8.上传博客

➤将.md后缀的博客文件放在……\blog\source_posts目录下，进入git界面

➤等待1~3分钟，即可在自己的博客上看到自己的文章

PS：写博客请遵照格式，不然上传容易报错。

1.1-Linux命令相关

2023-12-01

Linux命令相关

[root@localhost~]#——一个交互式的shell
exit—–退出当前用户
命名:ls—-查看已拥有文件，可执行文件，默认在ls后添加“ .”
ls /—–可以查看根目录
ls -a—-查看所有文件，包括隐藏文件
隐藏文件前缀会加上一个.
ls.. ——查看上一级目录
touch—-创建文件，方式：touch 文件名
ls -d——–查看目录
ls -al——–显示”所有”文件属性
文件属性： 开头d-目录，–文件，l-链接
后续每三个为一组：所有者+所有组+其他人
所有者：最大值为7，r（读）-4，w（写）-2，x（执行）-1，最大值为三者相加
r-读取，w-操作，x-执行
后续数学：显示当前文件数量
第一个root：当前用户名
第二个root：所属组
后续数字：文件大小
日期展示
. 代表在前一个目录，..代表上级目录
linux系统中只包括可执行文件和文本文件
mkdir——–创建目录
mkdir -p——创建多级目录
cd——更换目录—-cd.
cd..返回上一级目录
cd /返回根目录
tmp—-临时目录，var—日志，usr—一般用于放软件

/ 根目录：
根目录，文件的最顶端，整个文件系统的根目录

/bin 用户二进制文件：
存放系统所需的重要命令

/boot 引导加载程序文件
包含引导加载程序相关的文件（驱动）。
内核的initrd、vmlinux、grub文件位于/boot下。

/dev 设备文件：
包含设备文件。
这些包括终端设备、USB或连接到系统的任何设备

/etc 配置文件：
包含所有程序所需的配置文件。
也包含了用于启动/停止单个程序的启动和关闭shell脚本。

/home 目录
所有用户用home目录来存储他们的个人档案。

/lib 系统库
包含支持位于/bin和/sbin下的二进制文件的库文件.
（/lib 32位函数库
/lib64 64位库）

/media 可移动媒体设备
用于挂载可移动设备的临时目录。

/mnt 挂载目录
临时安装目录，系统管理员可以挂载文件系统。

/opt 可选的附加应用程序 有时候编译源码的时候可能需要
opt代表opitional；
包含从个别厂商的附加应用程序。
附加应用程序应该安装在/opt/或者/opt/的子目录下。

/proc 进程信息
虚拟的目录，是系统内存的映射。可以直接访问这个目录来获取系统信息

/root 临时设备的默认挂载点（”root”当前目录)

/sbin 系统二进制文件：
存放一些系统管理的命令，一般只能由超级权限用户root执行

/tmp 临时文件
包含系统和用户创建的临时文件。
当系统重新启动时，这个目录下的文件都将被删除。

/usr 用户程序
包含二进制文件、库文件、文档和二级程序的源代码。

/var 变量文件
var代表变量文件。
这个目录下可以找到内容可能增长的文件。

whoami——查看当前用户名
id——-查看当前用户权限
pwd—–查看当前所在目录
rm———删除命令—rm 文件名
rm -f ———-强制删除—– rm -f 文件名（比较危险）
rm -rf———强制递归删除文件夹内容以及文件夹
ls /bin|grep “mv”———筛选带“”关键字的文件
ls /bin|more———–查看文件
ls /bin|less————-上下查看文件（q退出）
cat————只能查看文本文件，查看可执行文件的话会出现乱码
vi———-编辑文件———-vi+文件名
编辑文件时，i键开始编辑文档，Esc+:+wq！保存文件，q!退出文件不保存。
cat -n———–查看文件时显示行号
chmod———-更改文件权限
echo——-输出内容到文件中——echo “内容” >>文件
find—–查找文件—-find 目录 -name（表示名字） 文件名—-通过文件名查找文件
find -user “root”———-查找用户
find size +/- 数字 ———-查找文件大小（+代表大于，-代表小于）
find -iname “”————查找文件名，不区分大小写，名字必须准确，PS：前后加*模糊搜索
