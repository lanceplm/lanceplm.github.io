路由相关

静态路由：路线固定
动态路由：根据最优解判断路线

动态ip：根据路由器dhcp分配ip，上线分配ip，下线回收ip
静态ip：手动设置ip

同时，静态优先

DHCP：:DHCP服务器负责动态分配IP地址、子网掩码和默认网关等网络配置信息。

TCP/IP模型：

127.0.0.1 本地回环地址

0.0.0.0 ====*匹配任意ip

TCP/IP模型共分为7层,从下到上依次是:

物理层(Physical Layer)定义了计算机之间传输数据的物理媒介,如网线,光纤等。

数据链路层(Data Link Layer)负责在物理层之上提供可靠的点到点连接,以太网、PPP等协议属于这一层。

网络层(Internet Layer)IP协议就属于这一层,负责地址和路由功能。

传输层(Transport Layer)提供端到端连接和传输控制,主要协议有TCP和UDP。

会话层(Session Layer)TCP/IP模型并没有明确定义会话层的协议规范。

表示层(Presentation Layer)TCP/IP模型并没有明确定义表示层的协议规范。

应用层(Application Layer)HTTP、FTP、SMTP等应用层协议都属于这个层次。

协议举例：

物理层:Ethernet(以太网)、RS-232、USB、光纤等。

数据链路层: Ethernet(IEEE 802.3)、PPP(点对点协议)、HDLC(高级数据链路控制)、ARP(地址解析协议)等。

网络层:IP(Internet协议)、ICNP(Internet控制消息协议)、IPSec(IP安全性协议)、OSPF(开放最短路径优先协议)等。

传输层:TCP(传输控制协议)、UDP(用户数据报协议)、SCTP(流控制传输协议)等。

会话层:NetBIOS(网络基本输入输出系统)、RPC(远程过程调用协议)、SSH(安全外壳协议)等。

表示层:ASCII码、JPEG(联合图像专家组)、MPEG(动态图像压缩标准)等。

应用层:HTTP(超文本传输协议)、FTP(文件传输协议)、SHTP(简单邮件传输协议)、DNS(域名系统)等。

子网掩码(Subnet Mask)是用于划分网络的一种技术，它与IP地址一起使用来确定网络中主机的地址范围。

路由表相关

每个路由器都有自己独立的路由表。

路由表是路由器内部维护的一个表,用于记录目的网络地址、下一跳路由器地址以及接口等信息,用于决定数据包的转发路径。

每个路由器的路由表内容是独立的,不会与其他路由器共享:

  • 路由表内容取决于路由器所处位置和邻居路由器信息。两台路由器可能位于不同网络位置,路由表内容不同。
  • 路由器通过不同的路由协议和静态路由配置学习和生成自己的路由表。
  • 即使同一供应商同型号路由器,如果网络位置和配置不同,路由表也会有差异。
  • 即使同一供应商路由器通过同一路由协议学习路由,由于网络拓扑和时延不同,两个路由表同步存在延迟。

所以总体来说:

  • 每个路由器都有自己独立的路由表
  • 路由表内容取决于路由器网络位置和配置信息
  • 不同路由器的路由表内容可能不同

这就保证了每个路由器都可以根据自身路由表正确转发数据包。

TCP协议

TCP是一种可靠的传输协议，在建立和终止连接时使用了三次握手和四次挥手的过程。

1.三次握手(Three-Way Handshake) :

a.第一步:客户端向服务器发送一个带有SYN(同步）标志的数据包，指示客户端要发起连接请求.

b.第二步:服务器收到客户端的请求后，回复一个带有SYN/ACK(同步/确认)标志的数据包，表示收到了请求，并同意建立连接。

c.第三步:客户端收到服务器的回复后，再发送一个带有ACK(确认）标志的数据包，表示已经收到服务器的同意，连接建立成功。

2四次挥手(Four-Way Handshake) :

a.第一步:当客户端要关闭连接时，发送一个带有FIN(结束）标志的数据包，表示不再发送数据。

b.第二步:服务器收到客户端的关闭请求后，回复一个带有ACK标志的数据包，表示已收到关闭请求。

c.第三步:服务器继续发送未完成的数据，将其发送给客户端。

d.第四步:服务器发送一个带有FIN标志的数据包，表示服务器的数据发送完毕，请求关闭连接。

e.第五步:客户端收到服务器的关闭请求后，回复一个带有ACK标志的数据包，表示已收到关闭请求。

f.第六步:客户端等待一段时间，确保服务器收到ACK回复后，关闭连接。

这样，经过三次握手和四次挥手的过程，TCP连接即建立和关闭完成。这些过程保证了可靠的数据传输和连接的安全关闭。

    ➤例如，src host 192.168.0.1 可以过滤出源 IP 地址为 192.168.0.1 的数据包。

    ➤例如，frame.len > 1000 可以过滤出长度大于 1000 字节的数据包。

    ➤例如，http.request.method == “GET” 可以过滤出 HTTP 请求方法为 GET 的数据包。

  9. 流量统计过滤规则：可以使用统计函数来过滤指定流量的数据包。

IP讲解

windows系统查询ip地址-cmd输入

ipv4每个位置命名

每个位置的最大可用地址为254即254.254.254.254，最大位置为255即255.255.255.255

d位.1表示网关，.255表示广播地址。

192.168.1.0是网络地址，192.168.168.255是广播地址。

192.168.1.1是一个IP地址。

具体来说:

  • 192.168.1.1是一个私有IP地址,它属于192.168.0.0/16网段内部的地址。
  • 192.168.1.1符合IPv4地址格式,共分为4段数字,每段范围在0-255之间。
  • 根据不同的上下文,192.168.1.1可以代表不同的地址类型:
    • 如果是单独的主机,那么它就是这个主机的IP地址。
    • 如果是子网,那么它可能是:
      • 子网的网络地址(很少情况)
      • 子网内第一个可用的主机地址
      • 子网的网关地址,路由器或交换机的IP地址
  • 一般来说,192.168.1.1最常见的情况是作为子网内第一个可用地址分配,也常作为子网的默认网关地址。

所以总结来说,192.168.1.1是一个私有IP地址,它可能代表单独主机的IP,也可能是子网内第一个地址或默认网关地址。

192.168.1.1/30是一个子网掩码为30位的子网。

子网掩码30位,表示:

  • 网络地址部分:192.168.1.0
  • 主机地址部分:2位(2^2=4个地址)

所以该子网可以分配的IP地址有:

  • 网络地址:192.168.1.0
  • 广播地址:192.168.1.3
  • 可用主机地址:192.168.1.1、192.168.1.2

总共可以分配2个IP地址给主机使用。

所以答案是:192.168.1.1/30子网可以分配2个主机IP地址。

a有8位，b有8位，c有8位，d应该有8位a+b+c+d=32

192.168.1.1/30 8+8+8+6=30,32-30=2,主机地址有两位（2^两=4个地址），分别减去网络地址(192.168.1.0)和广播地址(192.168.1.3)，可用地址剩下的是192.168.1.1、192.168.1.2

192.168.1.0不是一个合法的IP地址,它是网络地址,不能直接作为主机或网关使用。

正确的情况是:

  • 网络地址: 192.168.1.0
  • 第一个可用主机地址: 192.168.1.1
  • 网关地址: 192.168.1.1

理由如下:

  • 在IP地址分配中,网络地址是不可用的,它只表示子网本身。
  • 根据地址分配规则,第一个可用地址应该是网络地址加1,也就是192.168.1.1。
  • 网关地址一般设置为第一个可用主机地址,也就是192.168.1.1,以便路由转发。

所以您指出的正确,192.168.1.0是网络地址,不应该作为主机或网关使用。网关地址应该是第一个可用主机地址192.168.1.1。
